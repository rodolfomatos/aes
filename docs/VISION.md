# Vision

## Problem

LLM-assisted development suffers from:
- **No discipline** - LLMs produce code without structure
- **No validation** - untested, undocumented output
- **No accountability** - no traceability of decisions
- **No quality gates** - bad code slips through

Engineers using LLMs need a **protocol**, not just prompts.

## Solution

AES (Aggressive Engineering System) provides:

1. **Documentation-first** - VISION, PERSONAS, REQUIREMENTS, ROADMAP mandatory
2. **Hostile analysis** - Every task starts with questioning assumptions
3. **Surgical execution** - Minimal changes, maximum precision
4. **Goal-driven** - Success defined upfront, verified after
5. **Continuous validation** - `make check` enforces quality gates

Inspired by Karpathy's 4 principles, formalized into a repeatable protocol.

## Value

For **engineers**: Consistency, quality, predictability
For **teams**: Shared understanding, onboarding velocity, audit trail
For **projects**: Maintainable codebase, reduced tech debt, living docs

> "If it's not documented, it doesn't exist. If it's not tested, it's broken. If it's not questioned, it's wrong."
